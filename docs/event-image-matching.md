# Ручной реестр матчинга афиш событий

Дата сверки: 2026-03-19

## Правило

- Каноническое название события берём из [`festival_site_master.md`](/workspaces/kdg80/Исходные%20данные/festival_site_master.md).
- Ручная привязка афиши живёт в [`EVENT_IMAGE_MAP` в `festival.ts`](/workspaces/kdg80/site/src/lib/festival.ts#L125).
- Физический файл афиши берётся через ключ из [`media-manifest.json`](/workspaces/kdg80/site/src/data/media-manifest.json).
- Старые варианты заголовков из `v3` храним только в `alternateTitles`, чтобы переименование события не ломало карточку.
- Для горизонтальных афиш придерживаемся правила именования `тема - Фамилия_лектора`, но источником истины всё равно считаем manifest key, а не только имя файла.

## Что проверено отдельно руками

- У Анисимова причиной потери афиши был не файл и не manifest key, а устаревший `title` в `EVENT_IMAGE_MAP`.
- После переименований из `v4` матчинг нужно держать на канонических названиях мастер-файла, а не на старых `v3`-строках.
- Если афиша у существующего события снова пропадает, сначала сверять:
  1. название события в мастер-файле;
  2. `title` / `alternateTitles` в `EVENT_IMAGE_MAP`;
  3. наличие manifest key в `media-manifest.json`.
